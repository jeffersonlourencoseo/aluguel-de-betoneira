CORRECAO 2 — Breadcrumb Schema incompleto (JSON-LD)
---------------------------------------------------
PRIORIDADE: ALTA

Problema:
  O Schema BreadcrumbList em SchemaLocalBusiness.astro so renderiza 1 item:
  "Inicio" em TODAS as paginas.

  O Google recomenda breadcrumbs completos refletindo a hierarquia real:
  - Home: Inicio
  - /servicos: Inicio > Servicos
  - /servicos/betoneira-150l: Inicio > Servicos > Betoneira 150L
  - /blog: Inicio > Blog
  - /blog/artigo: Inicio > Blog > Titulo do Artigo
  - /sobre: Inicio > Quem Somos
  - /contato: Inicio > Contato

Acao:
  1. Abra src/components/SchemaLocalBusiness.astro
  2. Adicione uma prop opcional chamada `breadcrumbItems` (array de objetos
     com `name` e `url`) ou determine a hierarquia baseada na URL atual
     (Astro.url.pathname).
  3. Atualize o BreadcrumbList para renderizar dinamicamente todos os niveis.
     Exemplo de estrutura:
     ```
     {
       "@context": "https://schema.org",
       "@type": "BreadcrumbList",
       "itemListElement": [
         {
           "@type": "ListItem",
           "position": 1,
           "name": "Inicio",
           "item": "https://www.alugueldebetoneirarj.com.br/"
         },
         {
           "@type": "ListItem",
           "position": 2,
           "name": "Servicos",
           "item": "https://www.alugueldebetoneirarj.com.br/servicos/"
         }
         // etc
       ]
     }
     ```
  4. Em cada pagina que usa <SchemaLocalBusiness />, passe a hierarquia
     correta (ou faca o componente inferir automaticamente pelo pathname).
  5. Rode `npm run build` e valide o HTML gerado:
     - grep por BreadcrumbList em dist/ e confirme que paginas internas tem
       2+ itens.
  6. Teste a validade do JSON-LD no Google Rich Results Test
     (https://search.google.com/test/rich-results).

CORRECAO 3 — Tags do blog devem ser links filtraveis
------------------------------------------------------
PRIORIDADE: MEDIA

Problema:
  Em blog/index.astro, as tags sao renderizadas como <span> com cursor-default.
  Elas NAO funcionam como links. Isso e funcionalidade visual pura.

  Impacto: perda de navegacao interna e estrutura semantica. Usuarios tentam
  clicar e nada acontece.

Acao:
  1. Abra src/pages/blog/index.astro
  2. Localize onde as tags sao renderizadas (provavelmente um .map() de tags).
  3. Converta cada tag de <span> para <a> apontando para:
     /blog?tag={tag-slug}
     (ou crie rotas dinamicas /blog/tag/{tag} se preferir — mas query string
      e mais simples de implementar no Astro estatico).
  4. No topo da pagina blog/index.astro, leia a query string (Astro.url.searchParams)
     e, se houver ?tag=xxx, filtre os posts para mostrar apenas daquela tag.
     Se nao houver filtro, mostre todos (comportamento atual).
  5. Adicione um indicador visual quando um filtro esta ativo:
     "Mostrando artigos com a tag: {tag} [limpar filtro]"
  6. Estilize os links de tag para parecerem clicaveis (hover, underline, etc).
  7. Rode `npm run build` e confirme que nao ha erros.

CORRECAO 4 — Sitelinks Searchbox sem pagina de busca
----------------------------------------------------
PRIORIDADE: BAIXA (remover ou implementar)

Problema:
  O Schema WebSite em SchemaLocalBusiness.astro declara um SearchAction:
  ```
  "potentialAction": {
    "@type": "SearchAction",
    "target": "https://www.alugueldebetoneirarj.com.br/busca?q={search_term_string}",
    "query-input": "required name=search_term_string"
  }
  ```
  Mas NAO existe pagina /busca no projeto. Isso pode gerar erro no
  Google Search Console (URL nao encontrada).

Acao (escolha uma das duas):

  OPCAO A — Remover o SearchAction (mais rapido, menos risco):
    1. Em SchemaLocalBusiness.astro, remova o bloco potentialAction/SearchAction.
    2. Rode `npm run build` e confirme.

  OPCAO B — Implementar pagina de busca simples:
    1. Crie src/pages/busca.astro
    2. Leia a query string ?q=xxx
    3. Filtre posts do blog e/ou servicos que contenham o termo no titulo
       ou descricao.
    4. Mostre resultados em uma lista simples.
    5. Adicione um <input> de busca na pagina.
    6. Certifique-se de que a URL /busca?q={search_term_string} funciona.
    7. Rode `npm run build` e confirme.

  Diga ao usuario qual opcao voce escolheu e por que.
